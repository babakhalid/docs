## What to Include in Your Request

### For Best Results

Help us help you faster by including:

**1. Clear subject line**
```
Example: "Deployment failing with TypeScript error"
Example: "Cannot connect GitHub integration"
Example: "Question about token usage"
```

**2. Project information**
- Project name
- Project ID (find in Settings)
- Project URL (if deployed)

**3. Detailed description**
- What you were trying to do
- What you expected to happen
- What actually happened
- When did it start happening

**4. Steps to reproduce**
```
1. I clicked "Deploy" button
2. Selected "Deploy to Vercel"
3. Got error message: [error]
```

**5. Screenshots**
- Error messages
- Unexpected behavior
- Console errors (press F12)
- Build logs

**6. Environment details**
- Browser (Chrome, Firefox, Safari, etc.)
- Operating system (Mac, Windows, Linux)
- When did issue occur (date/time)

## Support Request Types

### Technical Issues

**Bug reports:**

When something isn't working correctly:
- Describe the bug clearly
- Include reproduction steps
- Attach screenshots of errors
- Share console logs if available
- Mention if it worked before

**Example email:**
```
Subject: Preview not loading after AI changes

Hi Yomake Support,

I'm having an issue with Project ID: proj_abc123
(https://my-site.yomake.ai)

Problem: After the AI makes changes, my preview shows a
white screen instead of updating.

Steps to reproduce:
1. Asked AI to "Add a hero section"
2. AI completed the changes
3. Preview shows white screen
4. Refresh button doesn't fix it

Browser: Chrome 120 on macOS Sonoma
Started happening: Today at 2pm EST

Screenshots attached.

Thanks!
```

---

### Integration Problems

**Connection or sync issues:**

For GitHub, Vercel, or Supabase problems:
- Which integration
- Error message received
- What you were trying to do
- Whether it worked before
- Screenshots of error

**Example email:**
```
Subject: GitHub sync failing - permission denied

Hi,

Cannot push to GitHub from Project ID: proj_xyz789

Error shown: "Permission denied - check GitHub app permissions"

I've verified:
- GitHub integration connected (green checkmark)
- Repository exists: username/my-project
- I'm the repo owner

Tried:
- Disconnecting and reconnecting
- Checking GitHub app permissions
- Different browser

Any ideas?
```

### Billing Questions

**Payment and subscription:**

For billing-related questions:
- Account email
- Issue description
- Invoice/receipt number (if applicable)
- Preferred resolution

**Example email:**
```
Subject: Charged twice for token purchase

Hi,

I purchased 35K tokens yesterday but was charged twice:
- Transaction 1: $30 at 10:15am
- Transaction 2: $30 at 10:16am

I only received one 35K token credit.

Account: user@email.com
Date: January 3, 2024

Could you please refund one charge or credit tokens?

Thank you!
```

---

### Account Issues

**Login or access problems:**

If you can't access your account:
- Account email address
- Last successful login
- Error message shown
- What you've tried

**Example email:**
```
Subject: Cannot log in - "Invalid credentials" error

Hi,

Cannot log into my account with email: user@email.com

Error: "Invalid credentials" even though I'm using
correct password from password manager.

Tried:
- Password reset link (not receiving email)
- Different browsers
- Clearing cache

Last successful login: December 30
Verified email is correct and has no typos

Please help!
```

## Follow-Up

### After Your Request

**What happens next:**

1. **Confirmation**: Auto-reply confirming we received your email
2. **Ticket creation**: Your request gets a ticket number
3. **Investigation**: Support team reviews your issue
4. **Response**: We reply with solution or questions
5. **Resolution**: Issue resolved, marked closed
6. **Feedback**: Optional survey about your experience

**Keep the same email thread** for follow-ups about the same issue.

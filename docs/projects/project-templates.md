# Project Templates

Start building faster with Yomake's professionally designed templates. This guide covers all available templates, what they include, and how to choose the right one.

---

## What Are Templates?

Templates are pre-built starting points for your projects. Each template includes:

- **React + TypeScript** - Type-safe, modern React
- **Tailwind CSS** - Utility-first styling framework
- **shadcn/ui** - Beautiful, accessible components
- **Vite** - Fast development server
- **Responsive design** - Mobile, tablet, desktop ready
- **Best practices** - Modern architecture patterns

Templates save you from starting with a blank canvas and provide a foundation you can build upon with the AI.

## Choosing the Right Template

### By Project Type

**Building a:**
- Personal site → Designer/Developer Portfolio
- Business site → Agency Website or Multi-Page
- Product launch → Product Launch or SaaS Landing
- Web app → Dashboard Layout or Blank
- Online store → Product Catalog + Checkout Flow
- Single product → Product Page
- Marketing page → Simple Landing or SaaS Landing

### By Experience Level

**Beginners:**
- Start with Simple Landing Page
- Or use Multi-Page Website for structure
- Avoid Blank template initially

**Intermediate:**
- SaaS Landing for feature-rich pages
- Dashboard Layout for app-like projects
- Portfolio templates for creative work

**Advanced:**
- Blank template for full control
- Dashboard for complex applications
- Checkout Flow for e-commerce

### By Timeline

**Need it fast (1-2 hours):**
- Simple Landing Page
- Product Launch Page
- Any portfolio template

**Medium build (3-5 hours):**
- SaaS Landing Page
- Agency Website
- Product Catalog

**Complex build (1-2 days):**
- Dashboard Layout
- Checkout Flow
- Multi-Page Website with many pages

## Best Practices

**Start with closest match**
- Choose template most similar to your goal
- Easier to modify than build from scratch
- AI understands template structure

**Don't overthink it**
- You can change everything
- Templates are suggestions
- Focus on getting started

**Build incrementally**
- Start with template
- Add features one by one
- Test as you go

**Keep it simple**
- Don't add every feature at once
- Launch with core functionality
- Iterate based on feedback
